JPA

JPA ajuda a mapear classes Java para tabelas de banco de dados. Com isso, uma classe pode representar uma tabela.

Anotações básicas

@Entity
public class Cliente {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long idLocal;

    private String nome;
}

@Entity

Indica que a classe será tratada como entidade no banco de dados.

@Id

Indica qual atributo é a chave primária.

@GeneratedValue

Indica que o valor do idLocal será gerado automaticamente.

Relacionamento ManyToOne

Usado quando muitos registros pertencem a um registro principal.

Exemplo: muitos produtos pertencem a uma categoria.

@ManyToOne
@JoinColumn(name = "categoria_id")
private Categoria categoria;

Relacionamento OneToMany

Usado quando um registro principal possui vários registros relacionados. 

@OneToMany(mappedBy = "categoria")
private List<Produto> produtos;

Exemplo completo

@Entity
public class Categoria {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long idLocal;

    private String nome;
}
@Entity
public class Produto {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long idLocal;

    private String nome;

    @ManyToOne
    @JoinColumn(name = "categoria_id")
    private Categoria categoria;
}

Buscando entidade relacionada

Quando um cadastro depende de outra entidade, normalmente buscamos primeiro a entidade relacionada pelo idLocal. 

Categoria categoria = categoriaRepository.findById(dto.getCategoriaId())
        .orElseThrow(() -> new RuntimeException("Categoria não encontrada"));

Produto produto = new Produto();
produto.setNome(dto.getNome());
produto.setCategoria(categoria);

return produtoRepository.save(produto);
