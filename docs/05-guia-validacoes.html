05 - Validações

Voltar ao início

Para que servem validações?

Validações impedem que dados incorretos sejam cadastrados no sistema. Elas ajudam a proteger a API contra informações incompletas ou inválidas.

Anotações comuns

Anotação Uso
@NotBlank Valida textos que não podem estar vazios
@NotNull Valida campos que não podem ser nulos
@Positive Valida números positivos
@Email Valida formato de e-mail
@Size Valida tamanho mínimo ou máximo

Exemplo


    @NotBlank(message = "O nome é obrigatório")
    private String nome,

    @NotNull(message = "O preço é obrigatório")
    @Positive(message = "O preço deve ser positivo")
    private Double preco

Usando @Valid no controller

Para que as validações funcionem, use @Valid. 


public Produto cadastrar(@Valid @RequestBody ProdutoRequestDTO dto)

Dependência necessária

O projeto precisa ter a dependência de validação no pom.xml. 

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>

Tratando erros de validação

Um handler pode capturar erros de validação e devolver mensagens mais organizadas. 

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map<String, String>> tratarValidacao(MethodArgumentNotValidException ex) {
        Map<String, String> erros = new HashMap<>();

        ex.getBindingResult().getFieldErrors().forEach(erro -> {
            erros.put(erro.getField(), erro.getDefaultMessage());
        });

        return ResponseEntity.badRequest().body(erros);
    }
}
