Camadas principais

Em uma API Spring Boot, é comum organizar o projeto em camadas. Cada camada tem uma responsabilidade.

Controller: recebe as requisições HTTP.
Service: concentra regras de negócio e lógica da aplicação.
Repository: faz a comunicação com o banco de dados.
Model: representa as entidades do sistema.
DTO: define os dados de entrada e saída da API.
Atenção: Os exemplos a seguir são apenas para exemplificar e demonstrar de forma simples classes, métodos e estruturas de um projeto. Eles não estão completos e caso sejam usados, necessitam adaptações.

Model

A model representa uma entidade do sistema. 

@Entity
public class Produto {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long idLocal;

    private String nome;
    private Double preco;

    // getters e setters
}

Repository

O repository acessa o banco de dados usando o Spring Data JPA. 

public interface ProdutoRepository extends JpaRepository<Produto, Long> {
}

Service

O service organiza a lógica do sistema. 

@Service
public class ProdutoService {

    @Autowired
    private ProdutoRepository repository;

    public List<Produto> listarTodos() {
        return repository.findAll();
    }

    public Produto buscarPorId(Long idLocal) {
        return repository.findById(idLocal)
                .orElseThrow(() -> new RuntimeException("Produto não encontrado"));
    }

    public Produto salvar(Produto produto) {
        return repository.save(produto);
    }

    public void deletar(Long idLocal) {
        repository.deleteById(idLocal);
    }
}

Controller

O controller expõe as rotas da API. 

@RestController
@RequestMapping("/produtos")
public class ProdutoController {

    @Autowired
    private ProdutoService service;

    @GetMapping
    public List<Produto> listar() {
        return service.listarTodos();
    }

    @PostMapping
    public Produto cadastrar(@RequestBody Produto produto) {
        return service.salvar(produto);
    }
}
